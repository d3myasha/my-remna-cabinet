# Frontend-only Cabinet

Упрощенная версия проекта: только фронтенд с локальной регистрацией без backend API.
В Docker-версии статика раздается через Caddy.

## Что делает

- Показывает страницу регистрации
- Проверяет имя, email, пароль и подтверждение пароля
- Сохраняет зарегистрированных пользователей в `localStorage`

## Запуск в разработке

```bash
npm install
npm run dev
```

Откройте `http://localhost:5173`.

## Запуск через Docker

```bash
cp .env.example .env
docker compose up -d --build
```

Откройте `http://localhost:3020` (или порт из `CABINET_PORT`).

## Переменные окружения

| Переменная | Описание | По умолчанию |
|---|---|---|
| `VITE_APP_NAME` | Название приложения | `Cabinet Frontend` |
| `VITE_APP_LOGO` | Короткий текст логотипа | `C` |
| `CABINET_PORT` | Порт Docker-контейнера | `3020` |

## Важно

Регистрация в текущем варианте работает только на клиенте: данные хранятся в браузере и не отправляются на сервер.
